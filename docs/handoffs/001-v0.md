## What this repo is

A harness that makes agent-generated CAx **verifiable without overclaiming**. Give it an
exported mesh; a cheap deterministic preflight catches the obviously-broken, an automatic
multi-view render is the primary gate, and a human makes the call.

One-line thesis: *coding agents write code well and reason about 3D space badly — so check
what geometry can cheaply prove, render the rest for a human in one glance, and keep the
model advisory.*

> If you ever see this described as "a deterministic verifier gates it against a
> pre-declared, machine-checkable `DesignSpec`, with a bounded VLM critique as a secondary
> pass" — **that is the killed design.** It failed its red-team. Plan §3.1.

## Build order (do not reorder)

1. **M1 — scaffold.** `pyproject.toml` (Python 3.12 exactly), `Makefile`, CI, governance,
   and the new `.claude/rules/verification-honesty.md`. No product code.
   *Done when:* `make validate` passes green on an empty package.
2. **M2 — the honest gate.** Inside M2: `preflight.py` first (pure, zero deps, test-first
   from fixtures), then `viewer/index.html`, `browser.py`, `render.py`, `report.py`.
   *Done when:* a human looks at a real part's render + verdict and decides.
3. **M3 — adoption.** Run it over so101 / i3mega exported STLs; wire one consumer.

**v0 takes an exported STL, not a build script.** The consumers already export STL, so v0
reaches a real user with no backend dispatch, no generation and no `DesignSpec`. That is the
whole reason the scope is this small.

**Do not build the deferred items** (generation loop, VLM annotation, slicer port, GUI, CAE)
without the trigger stated in plan §5. Each has one.

## Non-negotiables (do not regress)

Design — from the distilled red-team, plan §3.1:

- Green is *necessary, not sufficient*. Never emit a "verified/correct" verdict from
  deterministic checks alone. The word "verified" should not appear in the report.
- No bounded auto-critique loop with revert-on-break — it selects for gaming. The VLM (if
  ever built) annotates, never gates. "No model configured" is a first-class mode.
- Clearances are two-sided windows, never min-only.
- **Preflight alone is not a shippable gate.** Green-but-blind is the failure mode the
  red-team called worse than no gauge. That is why M2 is one milestone, not two.

Substrate — verified by reading polyfetch's source, plan §9:

- Use **`localhost`, not `127.0.0.1`**, for the viewer URL. The SSRF guard is literal-IP
  only: `localhost` passes, `127.0.0.1` raises, same destination. Wastes an hour if
  rediscovered.
- **Never assert three.js state via `page.evaluate`.** Patchright's isolated world makes
  page-script globals read `undefined` even when rendering succeeded. Screenshots are ground
  truth; `page.on(...)` answers "did it load". Highest-cost trap in the integration.
- Capture `pageerror` **and** `requestfailed` **and** non-200 `response`, not just
  `console` — parse errors never reach `console`. Force a failure once to confirm the
  listeners actually fire.

Process:

- **Stub-mode is mandatory.** Every external-tool wrapper degrades to an in-memory stub that
  logs what it would have done. This keeps CI green with no build123d/Chromium installed and
  is the primary test gate.
- **TDD, behavior-level, no Gherkin.** Skip trivial tests on declarative models.
- **Do not extract a shared CAD package** across so101/i3mega/this repo yet — two instances
  is not a stable pattern (plan §13.6).

## The headline test

The *silent boolean-fusion feature loss* fixture — a feature absorbed during fusion, still
manifold, volume barely shifted — must fail a volume-band or topology-count assertion. This
is the one real defect the cheap preflight uniquely catches. **If it passes green, the
preflight is worthless.** Build that fixture and test early.

## Traps

- **CAE research is owed and must not be faked.** `docs/outlook-cae.md` does not exist
  because the sweep failed twice. Plan §13.1 lists what to research. Writing it from memory
  would put unsourced claims in the corpus. Leave the gap.
- **Coordinate frames.** CAD-frame vs machine-frame `+Y` already bit `i3mega-pipettebot`
  once and became an enforced rule there. Adopt it on day one (plan §11).
- **Verify subagent findings before acting.** The scoping session had two research agents
  die on rate limits and one return off-task text that looked like a result.
- **Unverified inference — confirm at M2.** Plan §9.1 infers from a docs table that
  `check_ssrf` is not wired into polyfetch's `client.py`. That is inference, not a read of
  `client.py`.

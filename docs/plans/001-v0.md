**Design history in one line.** The original wedge — a pre-declared full `DesignSpec` as an
*authoritative correctness oracle*, with the VLM strictly subordinate — **failed its
red-team** (§3.1) and was replaced by the render-first design below. Do not resurrect it.

## 1. Context

**Problem.** Coding agents write code well and reason about 3D space badly. This is
structural, not a prompting deficiency — see [`../reference/landscape.md`](../reference/landscape.md)
§1. An agent asked to design a part cannot be trusted to look at a render and judge whether
it is right.

**Insight.** Do not ask the model to *see*. Make the geometry assert what it cheaply can,
render the rest fast for a human, and keep the model advisory.

**Estate position.** Two sibling repos independently built half this machinery and both
explicitly defer the loop itself. `so101-biolab-automation/docs/roadmap.md:47-58` documents
this exact loop as unbuilt future work and `:158-169` states no known project closes the
full loop.

**Outcome.** A standalone, domain-agnostic harness that makes CAx generation test-first.
`so101-biolab-automation` and `i3mega-pipettebot` are its first two consumers.

## 3. The approach

The honest primitive is not a machine oracle. It is **making the part cheap and fast to
judge, and honest about what it does not know.** Three layers, in this order of trust:

1. **Cheap deterministic preflight (necessary, not sufficient).** build-succeeds +
   manifold/watertight + bbox + mass, plus a browser-load check. Trustworthy for real
   *global/static* failures — including silent boolean-fusion feature loss, which is
   otherwise invisible. A gate against the obviously-broken, never a certificate of
   correctness.
2. **Auto-generated multi-view render as the PRIMARY gate.** For the defect classes the
   deterministic layer is blind to (a gusset over a hole, a hole that isn't through, a
   datum-offset mating feature), the real gate is a human looking at a fast, automatic
   3-view render. The harness's job is to make that glance instant, not to replace it.
3. **VLM as an assist, not an authority.** It annotates the render to draw the human's eye.
   It never gates, never auto-loops to "green", and cannot suppress human review. The system
   is fully functional with no model configured — which is why it is deferred out of v0
   entirely (§5).

**Why this ordering, and not the reverse.** The deterministic layer catches only
global/static/foreseen properties — roughly a third of real defects — and is blind to the
dominant class. Subordinating the render (the one signal that surfaces those) to a blind
gate is inverted. Worse, a blind-but-authoritative gate with revert-on-break **selects for
gaming**: a numerically-safe hack that dodges a check survives, while an honest fix that
risks a deterministic regression gets discarded. Hence: no auto-critique loop, no
auto-revert, human-driven iteration.

**`DesignSpec` survives only in a regression framing** — small and cheap (envelope, mass, a
few genuinely-mated clearances as two-sided fit *windows*), authored once and **reused
across many regenerations**: re-run after a build123d/OCCT bump, or a component reused
across assemblies. That is the only mode where pre-declaration earns its cost, so it is
post-v0 work (§5, M4). New checks are added **reactively** per the estate compound-learning
rule, never as a speculative tolerance/manufacturability DSL. Green is always documented as
*necessary, not sufficient*.

This dissolves most of the differentiation claim — fine, since positioning is estate-first
(§2). The value is a correct, honest internal instrument, not a moat.

### 3.1 The red-team, distilled

Reviewed 2026-07-20 by adversarial first-principles distillation; the kill case was run by
an **independent agent** in a separate context that had not seen the distillation.
Calibrated for an internal tool: moat/TAM objections voided, only frame-independent cuts
applied (correctness, Goodhart, ROI, rot, cold-start, real-defect coverage). Verdict:
**KILL-as-scoped**, reframe adopted. Full review text in git history —
`git show 2731ae3:docs/reviews/001-wedge-red-team.md`.

**What survived total concession**

- Deterministic checks are real and trustworthy for **global, static, foreseen** properties:
  build-succeeds, manifold/watertight, envelope, gross mass/volume, *foreseen* pairwise
  clearances, FDM overhang. Notably they catch **silent feature loss in boolean fusion** — a
  feature absorbed during fusion that still reports manifold, with volume inside a loose
  band. Genuinely valuable and otherwise invisible.
- A pre-declared spec earns its cost in exactly two modes: **regression across many
  regenerations**, and **offloading arithmetic humans eyeball badly** (tight clearance
  stacks, mass).

**What did not survive**

- **Full-taxonomy defect coverage — FALSE.** Realistic catch rate against the project's own
  example defects is roughly one third, and only for defects named *in advance*. The
  dominant class in agent-generated CAD is *local, unforeseen, sequential, or fidelity* —
  gusset overlapping a hole, a through-hole that isn't through, datum-offset mating feature,
  assembly-sequence blockage. Bbox/volume/manifold are structurally **blind** to all of
  these. Catching them needs a named per-feature assertion, which requires already knowing
  where the bug will be — circular.
- **Independence — FICTION as normally deployed.** One LLM authoring spec *and* geometry
  shares blind spots: correlated failure, not independent verification. Same for one human
  in one sitting.
- **"Green gauge" is worse than "no gauge" when framed as sufficient.** It licenses skipping
  the ten-second render glance for exactly the defects it cannot see, converting a
  would-have-been-caught defect into a false-confidence pass.

**Goodhart paths — the concrete gaming routes §12 tests against**

1. **One-sided clearance.** "clearance ≥ min" with no upper bound: shrink a part 0.5 mm
   everywhere → passes clearance, stays in bbox, ruins a press/snap fit. Hence clearances
   are two-sided *windows*, never min-only.
2. **Deletion beats repair.** Negative constraints are cheapest satisfied by removing the
   offending feature (delete the gusset) → manifold/bbox pass, weaker part scores greener.
3. **Mass band rewards hollowing or filling** to land in-band regardless of intent.

**Where the surviving value is real (ICP).** Not "any agent-generated part": a part
**regenerated many times against a stable, small acceptance contract** — an estate
labware/carriage component re-run after a build123d/OCCT bump, a fixture reused across
assemblies. For a one-shot generate-a-bracket, the render glance dominates.

**WINS IFF** the spec stays small and is reused across many regenerations; green is
documented as necessary-not-sufficient and never substitutes for viewing the render; new
checks are added reactively after a real recurring defect; clearances are two-sided; and
tolerance bands stay loose enough, with an owner, to survive OCCT/build123d/slicer drift.

**LOSES IFF** the spec is written once by the same intelligence that generates and framed as
a substitute for looking at the part; coverage is claimed as broad correctness rather than a
narrow static-property check; effort goes into a general tolerance/manufacturability DSL for
two hobby-scale consumers and the spec rots to a bbox+volume stub; clearance goes back to
min-only; or tight bands throw false failures across kernel bumps nobody maintains, so the
gate is quietly disabled while still implying verification.

v0 is the shaded path. Everything else is a named seam, not built.

```text
        |
        v

  sufficient              |  fail -> structured failure, surfaced to the human
        |                 |

        |
        v

        |

```

## 7. M2 — The honest gate (test-first)

### 7.1 Modules

| File | Responsibility |
|---|---|
| `src/caxgauge/verify/preflight.py` | `preflight(mesh) -> PreflightResult`. Checks: **build-ok** (input parses to a solid/mesh), **watertight/manifold**, **bbox within envelope**, **volume/mass within band**. Structured per-check pass/fail, explicitly labelled necessary-not-sufficient. Pure — no agent, no network |
| `viewer/index.html` | Static three.js + STLLoader. Loads a mesh from `?mesh=`; named camera presets (front/iso/top/right). On load failure it must **throw** (fire `pageerror`) — do not rely on a readable success flag (isolated-world caveat, §9) |
| `src/caxgauge/verify/browser.py` | `browser_load_check(mesh) -> BrowserResult`. Serve the viewer on **`localhost`** (§9.1), load via env-borrowed polyfetch `render_session`, capture `console_errors` + `network_failures` + `pageerror`. A mesh that won't parse is a **hard fail**. Stub/SKIP when polyfetch/Chromium absent |
| `src/caxgauge/verify/render.py` | Multi-view PNG capture through `polyfetch` at the named camera presets. **The primary gate** — makes the human glance instant |
| `src/caxgauge/report.py` | Assemble render + preflight verdict into one human review artifact. Green shown as *necessary, not sufficient*; never the word "verified" |

**Build order inside M2:** `preflight.py` first (pure, zero deps, fully testable from
fixtures), then `viewer/index.html`, then `browser.py`, then `render.py`, then `report.py`.
`browser.py` and `render.py` are the only polyfetch-dependent pieces — see the separability
note in §9.3.

### 7.2 Deterministic checks — implementation notes

- **Watertight/manifold**: dependency-free binary-STL integrity via `struct` (reuse the
  pattern in `../so101-biolab-automation/src/hardware/slicer/validate.py:98-184` and
  `../i3mega-pipettebot/tools/slicer/validate.py`), plus OCCT `is_valid()` when build123d is
  installed.
- **Silent boolean-fusion feature loss** (the headline value): caught deterministically only
  via a **topology-count / volume-band** assertion against expectation. M2 scope: a fixture
  where a feature was absorbed must fail either the volume band or a face/edge count
  assertion. Full manifest-diff regression is M4, not here.
- **bbox / volume**: compute from the mesh; assert within envelope/band. Two-sided where
  applicable.

### 7.3 Fixtures and acceptance

Fixture-driven, no agent, no VLM. Fixtures under `tests/fixtures/` (kept out of the
`.gitignore` mesh exclusion deliberately).

1. **Good part** — passes all preflight checks; browser-load clean.
2. **Non-manifold mesh** — fails the watertight check; other checks may still run.
3. **Feature-absorbed mesh** (silent fusion loss) — fails the volume-band or topology-count
   assertion. ***This is the headline test.*** If it passes green, the preflight is
   worthless.
4. **Oversize part** — fails the bbox/envelope assertion.
5. **Un-parseable mesh** — `browser_load_check` returns a hard fail via `pageerror`; assert
   the failure is caught (force the failure to confirm listeners fire, §9.2).
6. **Honesty test** — a part that passes preflight but is visibly wrong must NOT yield a
   "verified" verdict.
7. **Stub-mode CI (primary gate)** — full path runs with no build123d, no Chromium; asserts
   the result shape and the necessary-not-sufficient labelling; `SKIP`s tool-dependent legs
   cleanly.

Property tests (Hypothesis): a generated box of size X passes its own bbox assertion and
fails a deliberately perturbed one.

## 9. Substrate — `polyfetch-scrape`

Rendering geometry headlessly is the awkward part of M2. Native VTK/OpenGL offscreen
rendering needs GL/X11 in CI and is fragile. Instead: render in a **minimal static three.js
page** and drive it with `../polyfetch-scrape` (Apache-2.0, Python ≥3.11, v0.7.0), which
owns Patchright/Chromium install, launch, teardown, capture and an SSRF guard.

What that buys: multi-view capture via `render_session(url)`; VP8 `.webm` turntable video
(`Response.video_path`); viewport / orientation / device emulation; a deterministic
browser-side gate via `page.on(...)`; and E2E UI testing of the report itself.
`render_session()` capture is **always on** — `s.console_errors` and `s.network_failures`
fill for the whole session. One-shot `fetch()` needs opt-in
(`RenderOptions(capture_console=True, capture_network_failures=True)`). `aria_snapshot()` is
the current API; `page.accessibility.snapshot()` was removed upstream. This makes
`browser.py` mostly wiring, not new logic.

**Consumption:** env-borrow via `uv run --directory ../polyfetch-scrape` (see its
`USING.md`) — no venv poisoning.

### 9.1 Verified: use `localhost`, not `127.0.0.1`

Read directly at `../polyfetch-scrape/src/polyfetch_scrape/utils/_ssrf.py`. `check_ssrf(url)`
is **literal-IP only**: it parses the hostname and returns immediately if it is not a
literal IP. The module docstring states the scope — *"DNS names (incl. `localhost`) pass
through"*.

| URL | Result |
|---|---|
| `http://localhost:8080/viewer.html` | **Passes** — DNS name |
| `http://127.0.0.1:8080/viewer.html` | **Raises** — literal loopback IP |

Same destination, different outcome. This wastes an hour if rediscovered.

**Unverified inference — confirm empirically at M2.** Per `../polyfetch-scrape/docs/architecture.md`,
`check_ssrf` appears wired into `utils/sitemap.py` and `utils/discovery.py` only, not
`client.py` or the backends — so a direct `fetch()` / `render_session()` against a local
viewer is *probably* never guard-checked. That is inference from a doc table, not a read of
`client.py`.

### 9.2 Non-negotiable constraints

From `../polyfetch-scrape/AGENT_LEARNINGS.md`. Adopt from day one rather than rediscovering.

1. **Never assert three.js state via `page.evaluate`.** Patchright is a stealth Playwright
   fork running `evaluate` in an **isolated world**; page-script globals (`window.THREE`,
   module-scoped vars) read back `undefined` even when the scene rendered correctly. Their
   log records exactly this false negative. **Screenshots are ground truth for "did it
   render"; `page.on(...)` for "did it load".** Reserve `evaluate` for DOM you set or read
   structurally. This is the single highest-cost trap in the integration.
2. **Capture all four events.** Uncaught/parse errors fire `pageerror`, *not* `console`.
   Blocked or non-2xx resources fire `requestfailed` / non-200 `response`. Listening to
   `console` alone silently misses most real failures.
3. **A clean console means "no error on this network", not "no error".** Force a failure
   once and confirm the listeners fire.

## 12. Verification strategy

- **Fixture property tests** — §7.3.
- **Goodhart tests (from §3.1).** Assert the known gaming paths are *not* scored green: a
  part shrunk uniformly must fail a two-sided clearance window; a part with the offending
  feature deleted must not score better than the broken original — the report must surface
  the deletion, not hide it.
- **Necessary-not-sufficient honesty test.** Feed a part the preflight passes but that is
  visibly wrong (datum-offset hole). Assert the system **does not** report success — it
  presents the render for human decision and never emits a "verified/correct" verdict.
  Guards the false-confidence failure §3.1 calls worse-than-no-gauge.
- **Stub-mode CI (primary gate).** Runs end-to-end with no build123d, no OpenSCAD, no
  Chromium, no VLM; asserts the report is produced and green is labelled
  necessary-not-sufficient.
- **Golden-path integration (marker-gated).** One real part, real render capture.
- **Real acceptance test (M3).** The gauge runs over an existing part from
  `so101/src/hardware/parts.json` or `i3mega/tools/cad/parts.json` and produces a clean
  preflight + render a human accepts.
- **Regression test (M4).** Same part across a simulated kernel/tolerance shift: assert
  `verify/regression.py` surfaces the drift rather than silently passing or throwing a false
  failure.
